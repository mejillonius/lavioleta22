Accesibilidad en la Web

Descubre los conceptos básicos de accesibilidad en HTML

Es importante que diseñemos nuestro HTML teniendo en cuenta la accesibilidad.

Tener HTML accesible significa que las personas con discapacidad pueden utilizar la Web. Hay usuarios totalmente ciegos o con discapacidad visual, personas con problemas de pérdida auditiva y una multitud de otras discapacidades diferentes.

Desafortunadamente, a este tema no se le da la importancia que necesita y no parece tan interesante como otros.

¿Qué pasa si una persona no puede ver su página, pero todavía quiere consumir su contenido? Primero, ¿cómo lo hacen? No pueden usar el mouse, usan algo llamado lector de pantalla. No tienes que imaginarte eso. Puedes probar uno ahora: Google ofrece el Extensión ChromeVox Chrome. La accesibilidad también debe cuidar de permitir que las herramientas seleccionen elementos fácilmente o naveguen por las páginas.

Las páginas web y las aplicaciones web no siempre se construyen con la accesibilidad como uno de sus primeros objetivos, y tal vez la versión 1 no sea accesible, pero es posible hacer que una página web sea accesible después de ser construida. Cuanto antes, mejor, pero nunca es demasiado tarde.

Actualmente los sitios web creados por el gobierno u otras organizaciones públicas deben ser accesibles.

Las empresas que facturan más de 1.000.000 € tienen que disponer de sitios web accesibles.

¿Qué significa esto para hacer que un HTML sea accesible? Veamos algunos de los principales conceptos en losque se debe pensar.

Nota: Es imprescindible atender a otros ámbitos: En CSS, los colores, el contraste y las fuentes y tamaños. O cómo hacer que las imágenes SVG sean accesibles.

Utiliza HTML semántico

Desarrollar con HTML semántico es muy importante y es una de las principales ayudas que puedes ofrecer a personas con discapacidades y a buscadores. Veamos algunos escenarios comunes.

El contenido bien estructurado permite una navegación y un procesamiento más eficientes. Utiliza HTML y WAI-ARIA para mejorar la navegación y la orientación en páginas web y aplicaciones.

Regiones de página: identifica y marca regiones en páginas web utilizando roles HTML5 y WAI-ARIA.

Etiquetado de regiones:etiquetea regiones para permitir que los usuarios las distingan y accedan a ellas.

Encabezados: agrega encabezados y rellénalos lógicamente para etiquetar secciones de páginas web de acuerdo con sus relaciones e importancia.

Estructura del contenido: marca el contenido de una página de una manera que utilice elementos apropiados y significativos. <header>, <nav>, <main>, <footer>, <aside>, <section>, <article>, 

    <header>
      <img src="/images/logo.png" alt="SpaceBear Inc.">
    </header>
      <nav aria-label="Main Navigation">
    </nav>
    <nav aria-labelledby="quicknav-heading">
      <h5 id="quicknav-heading">Quick Navigation</h5>
    </nav>
    <main>
        <h1>Stellar launch weekend for the SpaceBear 7!</h1>
    </main>
      <aside>
      <h3>Related Articles</h3>
    </aside>
      <footer>
      <p>© 2021 SpaceBears Inc.</p>
      </footer>

Nota: Orientación sobre cómo crear sitios web que cumplan con las WCAG Maquetar Layout con estructuras semánticas

Es importante utilizar la estructura correcta para las etiquetas de encabezado. La más importante y única en cada html es <h1>, usa números más altos para los menos importantes, pero todos los encabezados del mismo nivel deben tener el mismo significado (piénsalo como una estructura de árbol)

           <h1>
              <h2>
                <h3>
              <h2>
              <h2>
                <h3>
                  <h4>

Usar <strong> y <em> en vez de <b> y <i>. Visualmente tienen el mismo aspecto, pero los dos primeros tienen más significado asociado a ellos. <b> y <i> son solamente visuales y están deprecados.

Las listas son importantes. Un lector de pantalla puede detectar una lista y proporcionar una descripción general, luego dejar que el usuario elija entrar en la lista o no.

Una tabla debe tener un <caption>: etiqueta que describe su contenido:

  <table>
    <caption>Dogs age</caption>
    <tr>
      <th>Dog</th>
      <th>Age</th>
    </tr>
    <tr>
      <td>Roger</td>
      <td>7</td>
    </tr>
  </table>

Nota: Especificaciones para el uso de las tablas con las WCAG Maquetar Layout con tablas accesibles.

Usa atributos alt para imágenes

Todas las imágenes deben tener un atributo alt que describe el contenido de la imagen. No es solo una buena práctica, es requerido por el estándar HTML y su HTML sin él no está validado.

<img src="dog.png" alt="My dog" title="Another dog">

También ayuda a los motores de búsqueda a posicionar, puede ser un incentivo para que lo agregues.

Nota: Orientación sobre diferentes especificaciones para el uso de las imágenes con las WCAG Maquetar Layout con imágenes accesibles , <figure>, <figcaption>, longdesc, aria-describedby, <picture>

Utiliza el atributo role

Los atributos role Permiten asignar roles específicos a los diversos elementos de su página.

Puedes asignar muchos roles diferentes: complementario, lista, elemento de lista, principal, navegación, región, pestaña, alerta, aplicación, artículo, banner, botón, celda, casilla de verificación, información de contenido, diálogo, documento, fuente, figura, formulario, cuadrícula, cuadrícula, encabezado, img, cuadro de lista, fila, grupo de filas, búsqueda, cambiar, tabla, panel de pestañas, cuadro de texto, temporizador.

Son muchos los atributos role y para que investigues, adjunto un vínculo a la referencia completa de cada uno de ellos este enlace MDN. Pero no es necesario asignar un rol a cada elemento de la página. Los lectores de pantalla pueden inferir de l etiqueta HTML en la mayoría de los casos. Por ejemplo, no es necesario agregar una etiqueta de función o rol a etiquetas semánticas como <nav>, <header>, <figure>, <picture>, <footer>, <main>, <aside>, <section>, <article>, <label>.El siguiente ejemplo de etiqueta semántica muestra un uso mas eficaz con etiquetas semánticas y no es necesario usar el atributo role , ni es recomendable. 

<nav> <ul> <li><a href="/">Home</a></li>
              <li><a href="/blog">Blog</a></li> </ul> </nav>

Si te ves forzado a usar una etiqueta <div>en lugar de <nav>, puedes usar el atributo role="navigation" . 

<div role="navigation"> <ul><li><a href="/">Home</a></li>
            <li><a href="/blog">Blog</a></li> </ul> </div>

Así que aquí tienes un ejemplo práctico: role se utiliza para asignar un valor significativo cuando la etiqueta no transmite ya el significado.

Utiliza el atributo tabindex

Los atributos tabindex permiten cambiar el orden en que al presionar la tecla Tab se seleccionan los elementos "seleccionables". De forma predeterminada, solo los enlaces y elementos de formulario se pueden "seleccionar" mediante la navegación con la tecla Tab (y no es necesariobconfigurar tabindex en ellos).

Añadiendo tabindex="0" convertimos un elemento HTML, seleccionable: 

<div tabindex="0">
              ...
            </div>

Usando tabindex="-1" en su lugar, eliminas el elemento de esta navegación basada en pestañas, puede ser bastante útil.

Utiliza atributos aria

ARIA es un acrónimo que significa Accessible Rich Internet Applications y define la semántica que se puede aplicar a los elementos.

aria-label

Este atributo se usa para agregar una cadena para describir un elemento, cuando la etiqueta no sea la correcta o no sea descriptiva.

Ejemplo:

<p aria-label="The description of the product">...</p>

Utiliza este atributo en un elemento o etiqueta etiqueta no explícita, si es necesario.

aria-labelledby

Este atributo establece una correlación entre el elemento actual y el que lo etiqueta.

Si sabes como un input se puede asociar a un label, estye atributo funciona de igual manera cuando sea necesario

Pasamos la identificación del elemento que describe el elemento actual.

Ejemplo:

<h3 id="description">The description of the product</h3>
  
<p aria-labelledby=“description”>
  …
  </p>

aria-describedby

Este atributo nos permite asociar un elemento con otro elemento que sirve como descripción.

Ejemplo:

<button aria-describedby="payNowDescription" >Pay now</button>
  
<div id=“payNowDescription”>Clicking 
    the button will send you to our Stripe form!</div>

aria-hidden

Usa aria-hidden para ocultar contenido

Me gustan los diseños minimalistas. Si el documento es principalmente contenido, con algunos enlaces, pero algunos de los contenidos son solo elementos visuales que no suman a la experiencia de una persona que no puede ver la página. Como logotipos, imágenes: Añadiendo el atributo aria-hidden le dirá a los lectores de pantalla que ignoren ese elemento.
